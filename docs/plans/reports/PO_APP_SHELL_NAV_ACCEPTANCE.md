## Проверенные сценарии
- [x] На `/`, `/upload`, `/dev-tools` видны одни и те же 3 пункта навигации (Дашборд, Загрузка файлов, Dev Tools)

## Что хорошо
- Единый AppShell на всех основных маршрутах — пользователь всегда видит одинаковую шапку и может переключаться между разделами без «потери» навигации
- Активная вкладка заметна: фон + жирный шрифт; для скринридеров есть `aria-current="page"`
- Подписи понятны: «Дашборд», «Загрузка файлов», «Dev Tools» — сразу ясно, куда ведёт каждый пункт
- Визуальные разделители `|` между пунктами улучшают читаемость
- Страницы не дублируют Header локально — нет «двойной шапки»
- Навигация не зависит от API/БД: работает даже при недоступном backend (важно для демо)

## Что не устраивает

Замечаний, блокирующих приёмку, нет.

### Наблюдение (minor, вне scope задачи)
**Где:** `Header` — блок `header-nav`  
**Почему может мешать:** на экранах уже `md` (<768px) пункты навигации скрыты (`hidden md:flex`), альтернативного меню нет  
**Критичность:** minor — для демо на десктопе не блокирует; стоит учесть в отдельной mobile-задаче
