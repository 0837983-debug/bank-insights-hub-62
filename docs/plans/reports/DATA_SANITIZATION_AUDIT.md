## 3) Отдельная проверка рисков real-data

### 3.1 Риск в `ods.*`

- Выявлены крупные суммы и профильные банковские статьи:
  - диапазон `ods.balance.value`: `-5000000000.00 .. 6800000000.00`
  - диапазон `ods.fin_results.value`: `-45000000.0000 .. 125000000.0000`
- Примеры строк включают доменные статьи `АКТИВЫ`, `ПАССИВЫ`, `КАПИТАЛ`, `ЧПД`, `ЧКД`.
- Риск: **высокий** (реалистичный финансовый профиль данных).

### 3.2 Риск в `stg.*`

- Данные staging по содержимому и диапазонам совпадают/близки к ODS.
- Риск: **высокий** (дублирование бизнес-данных перед загрузкой).

### 3.3 Риск в `ing.uploads`

- Таблица содержит `filename`, `original_filename`, `file_path`, статус и метаданные.
- Встречаются имена файлов с банковским контекстом и русскоязычные оригиналы (включая Excel).
- Риск: **средний/высокий** (метаданные источников и названия исходников).

### 3.4 Риск в `log.*`

- `log.upload_errors` в текущем срезе пуста (`0` строк), но структура включает `field_value`, `error_message`.
- Риск: **потенциальный средний** (при наличии строк может храниться чувствительный контент).

## 4) Отдельная проверка репозитория (дампы/CSV/XLSX)

Проверены файловые артефакты форматов CSV/XLSX/XLS и архивы:

- `backend/row/processed`: 111 файлов данных (`62` CSV + `49` Excel)
  - есть файлы с явно доменными именами и временными метками
  - есть файлы с русскоязычными названиями, включая "Средний баланс ..."
- `test-data/uploads`: 19 CSV (синтетический тестовый набор для dev/test)

- явные SQL-dump файлы в корне/подкаталогах не выявлены.

Оценка риска репозитория:

- `backend/row/processed/**/*.{csv,xlsx,xls}` -> **высокий риск** попадания реальных/полуреалистичных исходников в handoff.
- `test-data/uploads/*.csv` -> **допустимый риск**, оставить как тестовый dataset.
- `docs.zip` -> **низкий риск утечки бизнес-данных**, но есть организационный шум (архив и macOS metadata).
