## Схемы (owners) в `fxldp_full_ddl.sql`

DDL содержит следующие схемы (owner) и примерный “вес” по таблицам:

- **`TBG` (~4333 таблицы)**: основная бизнес‑схема АБС (клиенты, счета, операции, справочники, интеграции)
- **`TBGMIGR` (~505 таблиц)**: миграционные/временные/служебные таблицы для переноса/синхронизации
- **`FXL_CUST` (~410 таблиц)**: интеграционный/канальный контур (в т.ч. “витрина” клиентов/счетов для фронта)
- **`FXLAPI` (~14 таблиц)**: API/очереди/интеграция (встречаются AQ_* таблицы)
- **`ORDS_METADATA` (~27 таблиц)**: метаданные Oracle REST Data Services (OAuth clients/roles и т.п.)
- **Системные Oracle‑схемы**: `SYS`, `SYSTEM`, `WMSYS`, `LBACSYS`, `AUDSYS`, `MDSYS` — **не** бизнес‑данные АБС, а инфраструктура БД
- **Прочее**: `DT_ADM`, `SHAPOVALOV_DBA`, `VBFXL_IFR` — локальные/админские/вспомогательные

### Практическая рекомендация по поиску бизнес‑данных

- Если вы ищете **источник истины** по клиентам/счетам — начинайте с **`TBG`**.
- Если вы ищете “как это видит фронт/каналы” — смотрите **`FXL_CUST`**.
- Если видите `SYS/SYSTEM/...` — почти всегда это **служебное**, не предметная область АБС.
