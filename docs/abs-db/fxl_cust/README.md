## FXL_CUST — структура витрины/канального контура

Источник: `fxldp_full_ddl.sql`.

### Объем

- **Tables**: 410
- **Views**: 128

### Что это за слой

`FXL_CUST` выглядит как **витрина/канальный контур**:

- есть “ядро” сущностей (`FX_*`: клиент, счет, договор, документы, пользователи/роли)
- есть “прикладная обвязка” и интеграции (`FFB_*`, `GATE$*`, `AQ$_*`)
- много “временных/исторических/технических” таблиц (суффиксы с датами, `*_TMP`, `*_BAK_*`, `*_HIST`, `*_OLD`)

Важно: в DDL **нет внешних ключей из `FXL_CUST` в `TBG`**, поэтому консистентность с ядром АБС обеспечивается не FK, а загрузками/процедурами/интеграцией. При этом многие **views** в `FXL_CUST` могут “подтягивать” данные из `TBG` (как витринные представления).

### Файлы

- `object-list.md`: полный список таблиц и view, сгруппированный по префиксам/подкластерам
- `clusters.md`: смысловые кластеры и “что за информация” в каждом
- `key-entities.md`: детализация ключевых сущностей (`FX_CLIENT`, `FX_ACCOUNT`, `FX_DOC`, …) и основных витринных view (`FFB_CUSTOMERS`, `FFB_ACCOUNTS`, …)
