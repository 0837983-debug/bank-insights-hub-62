# Быстрый старт

Это руководство поможет вам запустить проект за несколько минут.

## Предварительные требования

Убедитесь, что вы выполнили [установку](/getting-started/installation).

## Запуск проекта

### Вариант 1: Использование скрипта (рекомендуется)

```bash
./scripts/start-servers.sh
```

Этот скрипт автоматически запустит оба сервера (backend и frontend).

### Вариант 2: Ручной запуск

**Терминал 1 - Backend:**
```bash
cd backend
npm run dev
```

**Терминал 2 - Frontend:**
```bash
npm run dev
```

## Проверка работы

1. **Backend API** должен быть доступен на `http://localhost:3001`
   - Проверьте: `curl http://localhost:3001/api/health` (если endpoint существует)
   - Или откройте: `http://localhost:3001/api/data?query_id=kpis&component_Id=kpis&parametrs={}`

2. **Frontend** должен быть доступен на `http://localhost:8080`
   - Откройте в браузере: `http://localhost:8080`

## Первые шаги

3. Изучите структуру дашборда

## Добавление нового элемента (карточки или таблицы)

Чтобы на дашборде появилась **новая карточка KPI** или **новая таблица**, элементы добавляются через конфигурацию в БД — код фронтенда менять не нужно.

### Краткая последовательность

1. **Данные в БД** — в схеме `mart` должны быть данные (для KPI — view/таблица, для таблицы — таблица с полями иерархии и метрик).
2. **Конфиг запроса** — запись в `config.component_queries` с `query_id`, по которому SQL Builder строит запрос к данным.
3. **Компонент** — запись в `config.components` с уникальным `id`, типом `card` или `table`, полем `data_source_key` = `query_id` из п. 2.
4. **Для таблицы дополнительно:** колонки в `config.component_fields` (или в JSON колонок компонента в layout), при необходимости форматы в `config.formats`.
5. **Привязка к layout** — добавление компонента в нужную секцию через `config.layout_component_mapping` (или через view/функцию, формирующую layout).
6. **Проверка** — перезагрузить дашборд; данные запрашиваются по `data_source_key`, колонки и форматы берутся из layout.

Даты для таблиц передаются с дашборда из header (`query_id=header_dates`); параметры дат подставляются в запрос автоматически.

**Подробная пошаговая инструкция с примерами SQL:** [Добавление нового источника данных](/guides/adding-data-source).

## Решение проблем

Если что-то не работает:

1. Проверьте, что база данных запущена и доступна
2. Убедитесь, что все зависимости установлены
3. Проверьте переменные окружения в `backend/.env`
4. Посмотрите [руководство по решению проблем](/guides/troubleshooting)
