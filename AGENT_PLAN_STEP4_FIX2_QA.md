# План: Шаг 4 (фикс 2) — QA — Проверка header_dates и таблицы

**Цель:** проверить, что header_dates возвращает корректные даты и таблица отображается.
**Статус:** ⏳ Ожидает начала

## Задачи
- [ ] Проверить GET /api/data/header_dates возвращает три даты:
  - periodDate = последний день предыдущего месяца
  - ppDate = последний день месяца перед periodDate
  - pyDate = последний день года перед periodDate
- [ ] Проверить формат ответа: `{ componentId, type, rows }`
- [ ] Проверить, что таблица "Активы" отображается на фронте
- [ ] Проверить, что данные в таблице корректные
- [ ] Прогнать все предыдущие автотесты (регрессия)
- [ ] Если есть фронт-тесты — запустить их тоже

## Тестовые сценарии

### API тест
```bash
curl "http://localhost:3001/api/data/header_dates?component_id=header"
```
Ожидаемый ответ:
```json
{
  "componentId": "header",
  "type": "table",
  "rows": [{
    "periodDate": "2025-12-31",
    "ppDate": "2025-11-30",
    "pyDate": "2024-12-31"
  }]
}
```

### UI тест
- Открыть http://localhost:8080
- Проверить, что секция "Баланс" содержит таблицу "Активы" с данными
- Проверить, что нет сообщения "Нет данных для таблицы"

## Критерии завершения
- [ ] API возвращает корректные даты
- [ ] Таблица отображается
- [ ] Регрессия не сломалась
