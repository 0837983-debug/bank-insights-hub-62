# План: Шаг 4 (фикс 4) — QA — Проверка URL

**Цель:** проверить, что URL формируется правильно и данные загружаются.
**Статус:** ⏳ Ожидает начала

## Задачи
- [ ] Открыть http://localhost:8080
- [ ] Открыть DevTools → Network tab
- [ ] Проверить, что запросы к `/api/data/header_dates` идут с правильным URL
- [ ] Проверить, что нет ошибки "Route not found"
- [ ] Проверить, что таблица "Активы" отображает данные
- [ ] Прогнать все предыдущие автотесты (регрессия)

## Ожидаемые URL в Network tab
- `http://localhost:3001/api/data/header_dates?component_id=header` — должен возвращать 200
- `http://localhost:3001/api/data/assets_table?component_id=assets_table&p1=...` — должен возвращать 200

## Критерии завершения
- [ ] URL правильный (без дублирования)
- [ ] Нет 404 ошибок
- [ ] Таблица отображает данные
- [ ] Регрессия не сломалась
